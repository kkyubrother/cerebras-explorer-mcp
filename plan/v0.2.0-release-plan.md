# cerebras-explorer-mcp v0.2.0 Release Plan

작성일: 2026-05-12 KST
대상: `cerebras-explorer-mcp@0.1.0`
기준 커밋: `b12c0b1838260526977587aab33b33b26ddca275`

## 목표

`0.1.0`의 read-only 코드 탐색 서버 구조를 유지하면서, v0.2.0에서 클라이언트 표면 호환성, 시크릿 유출 방지, 설치 문서, stdio 안정성을 릴리스 가능한 수준으로 끌어올린다.

핵심 기준은 다음과 같다.

- read-only MCP 툴이라는 신호를 클라이언트에 정확히 제공한다.
- `.env`, SSH key, cloud credential 등 비밀 파일과 주요 시크릿 문자열이 tool result로 새지 않게 한다.
- 내부 repo tool 결과가 다음 provider 호출로 넘어가기 전에 비밀 문자열이 차단 또는 redacted 되도록 한다.
- stdio JSON-RPC 채널이 stdout 오염으로 깨지지 않도록 회귀 테스트를 둔다.
- README와 integration 예제를 신규 사용자가 5분 안에 연결할 수 있는 형태로 정리한다.
- 범위가 큰 protocol 확장 작업은 v0.2.0 필수 게이트에서 분리한다.

## 릴리스 게이트

v0.2.0 final 태그 전까지 아래 항목은 모두 완료되어야 한다.

- 9개 노출 툴 전체에 read-only `annotations` 선언
- secret deny-list와 핵심 redaction 적용
- stdio purity 가드와 child process 회귀 테스트 추가
- Gemini CLI integration 문서 추가
- Codex CLI example 갱신
- README 슬림화, Security 단락 추가, 노출 툴 구성 명시, `#main` 참조 정정
- 신규 테스트가 `npm test`에서 실제로 실행되도록 test runner 보정
- README, DESIGN, integrations 문서가 변경된 공개 계약과 서로 일치
- 신규 runtime dependency 없음
- CHANGELOG 작성
- RC 태그와 dogfood 후 final 릴리스

## 구현 원칙

### 테스트 실행 범위

현재 `package.json`의 `npm test`는 `node --test ./tests/*.test.mjs` 형태이므로 `tests/security/`나 `tests/integration/` 아래 신규 테스트는 실행되지 않는다.

둘 중 하나를 선택한다.

- 신규 테스트를 `tests/*.test.mjs` 최상위에 둔다.
- 또는 test script를 재귀 실행 형태로 바꾼다.

완료 기준은 `npm test` 출력에서 신규 security, stdio, annotations 테스트가 실제로 실행되었음을 확인하는 것이다.

### Zero-dependency 유지

DESIGN의 설계 제약에 따라 Node 표준 라이브러리를 우선한다. secret glob matcher, redaction, stdio 테스트, integration 문서 작업은 새 runtime dependency 없이 구현한다. 새 패키지가 꼭 필요하면 v0.2.0 범위 밖 별도 결정으로 넘긴다.

### Secret deny-list 적용 범위

`readFile`과 `grep`만 막으면 충분하지 않다. 최소한 다음 경로가 같은 정책을 타야 한다.

- file traversal: `walkFiles`
- ripgrep fast path: deny pattern을 `--glob` exclude로 전달
- fallback grep
- `RepoToolkit.readFile`
- symbol/context enrichment
- `runtime.mjs`의 `readEvidenceSnippet()`

`readEvidenceSnippet()`은 `RepoToolkit.readFile`을 거치지 않고 직접 파일을 읽을 수 있으므로 별도 보강이 필요하다.

fixture는 최소한 다음 케이스를 포함한다.

- `.env`, `.env.local`
- `.npmrc`
- `.ssh/id_rsa`
- `.aws/credentials`
- `secrets/app.pem`
- `src/credentials.json`
- 일반 소스 파일 또는 git patch 안에 포함된 API key, JWT, private key block

완료 기준은 user-facing MCP 응답뿐 아니라 provider-facing tool message에도 원문 secret이 포함되지 않는 것이다.

### Redaction 모델

`groundingStatus: "redacted"`를 추가하는 방식보다 redaction 여부를 별도 필드로 분리한다.

권장 형태:

```json
{
  "redacted": true,
  "redactions": ["openai-api-key", "jwt"]
}
```

문자열 치환 표기는 코드베이스 기본 스타일에 맞춰 ASCII 형태를 우선한다.

```text
[REDACTED:openai-api-key]
```

redaction은 다음 surface를 포함한다.

- `content[0].text`
- `structuredContent`
- `explore` Markdown
- `explore_v2` Markdown
- evidence snippet
- git diff/show patch
- `_debug` 문자열 필드
- provider-facing tool message

line reference는 유지하고 민감 문자열만 치환한다. redaction metadata는 README와 DESIGN의 반환 계약에 additive field로 문서화한다.

secret deny-list 비활성 옵션을 제공하는 경우 README Security 단락에 테스트/로컬 디버그 전용이며 기본값은 안전 모드라는 경고를 둔다.

### Gemini CLI 명령

Gemini CLI 문서 스타일에 맞춰 아래 형태를 권장한다.

```bash
gemini mcp add -e CEREBRAS_API_KEY="$CEREBRAS_API_KEY" cerebras-explorer npx -- -y github:kkyubrother/cerebras-explorer-mcp#v0.1.0
```

`CEREBRAS_API_KEY`는 Gemini CLI의 environment sanitization에 걸리므로 server config의 `env` 블록에도 명시한다.

### v0.2.0 범위 통제

`roots/list`, `resource_link`, provider fallback 같은 작업은 가치가 있지만 회귀 범위가 크다. v0.2.0 final의 필수 게이트가 아니라 v0.2.x 후보로 둔다.

README의 8KB 제한은 하드 게이트가 아니라 슬림화 목표로 둔다. 공개 계약, 보안 경계, tag 기반 install 안내, 세션/반환 스키마가 사라지지 않도록 README 또는 DESIGN/docs에 보존한다.

## 완료 기준

v0.2.0은 다음 상태일 때 릴리스한다.

- 기본 8개, 최대 9개 노출 툴 구성이 README와 실제 `tools/list`에서 일치한다.
- 모든 노출 툴이 read-only annotations를 가진다.
- `.env`, `.ssh/id_rsa`, `.aws/credentials`, `secrets/**`가 read, grep, snippet 경로에서 노출되지 않는다.
- `.env.local`, `.npmrc`, `*.pem`, `credentials.json`, git diff/show patch의 secret도 차단 또는 redacted 된다.
- 주요 API key, PAT, JWT, private key block이 user-facing result와 provider-facing tool message에서 redacted 된다.
- stdout에는 NDJSON과 Content-Length framing 모두에서 JSON-RPC frame 외 출력이 없다.
- Claude Code, Codex, OpenCode, Cursor, Continue, Claude Desktop, Gemini 예제가 서로 모순되지 않는다.
- README와 integrations의 `github:` install spec이 존재하는 tag 또는 branch를 가리키며 `#main` 불일치가 없다.
- README와 DESIGN이 `redacted`/`redactions`, secret deny-list, Security model을 같은 방향으로 설명한다.
- `npm ls --depth=0` 기준 신규 runtime dependency가 없다.
- `npm test`가 신규 회귀 테스트를 포함해 통과한다.
